# 외부 해킹 위험성 분석

> 집합론적 사고 × Gary Klein RPD 관점
> 작성: 2026-03-19

---

## 1. 집합론적 관점

### 공격 표면(Attack Surface) = 여러 집합의 합집합

```
U (전체 위협 공간)
│
├── A = 인터넷에 노출된 서비스
│     → NanoClaw 자체: 포트 없음, 외부 노출 없음
│     → Tailscale: VPN이므로 공개 IP 없음
│     → 텔레그램 봇: Telegram 서버가 중간에서 차단
│
├── B = 인증 탈취 가능 경로
│     → .env 파일 (OAuth 토큰, 봇 토큰)
│     → macOS 계정 비밀번호
│     → iCloud/Apple ID
│
├── C = 코드 실행 경로
│     → NanoClaw 프로세스
│     → Docker 컨테이너
│     → 마운트된 폴더를 통한 파일 조작
│
└── D = 공급망(Supply Chain)
      → npm 패키지 (grammy 등 의존성)
      → Docker 이미지
      → brew 패키지
```

### 핵심: A ∩ B ∩ C = 실제 위험 교집합

지금 설정에서 **A(외부 노출)가 매우 작음** — 이게 핵심 방어선.

```
실제 위험 교집합:

  [외부노출] ∩ [인증탈취] ∩ [코드실행]
       ↓
  현재: A가 거의 공집합에 가까움
  → 전체 교집합도 작아짐
```

**그러나 D(공급망)는 A와 독립적으로 위험**
- `npm install` 한 번에 수백 개 패키지가 들어옴
- 외부 노출 여부와 무관하게 위협이 됨

## 2. Gary Klein RPD 관점

RPD(Recognition-Primed Decision)는 전문가가 상황을 패턴으로 인식하고 첫 번째 그럴듯한 행동을 실행하는 모델. **공격자의 RPD**로 분석.

### 공격자가 타깃으로 인식하는 패턴

**패턴 1: "텔레그램 봇 운영자"**
- 봇 토큰 유출 시 → 봇 메시지 도청, 발송 가능
- 단, 서버 침투는 불가 (봇 토큰 ≠ 서버 접근)

**패턴 2: "로컬 AI 에이전트 운영자"**
- 공격자 입장에서 매력적인 타깃이 아님
- 외부 포트가 없으므로 자동화 스캔에 안 걸림

**패턴 3: "개발자 맥미니"**
- npm/brew 패키지를 통한 공급망 공격
- 개인 특정 공격이 아닌 무차별 공격
- 현실적으로 가장 높은 확률

### 공격자의 행동 스크립트(Action Script)

| 시나리오 | 발동 조건 | 현실성 |
|---------|----------|--------|
| 포트 스캔 후 침투 | 열린 포트 발견 | **낮음** — 외부 포트 없음 |
| .env 파일 탈취 | 파일시스템 접근 획득 후 | **중간** — 다른 취약점 먼저 필요 |
| 봇 토큰 도청 | 네트워크 중간자 공격 | **낮음** — HTTPS |
| npm 악성 패키지 | 패키지 설치 시 | **중간** — 업계 전반 위협 |
| 프롬프트 인젝션 | 에이전트가 악성 파일 읽을 때 | **중간** — 현실적 시나리오 |
| 피싱/소셜엔지니어링 | 본인이 속을 때 | **높음** — 기술과 무관 |

## 3. 종합 판단

### 현실적 위협 순위

```
1위 (높음)   : 본인 실수 — 잘못된 마운트, 에이전트 오작동
2위 (중간)   : 공급망 공격 — npm/brew 악성 패키지
3위 (중간)   : 프롬프트 인젝션 — 마운트 폴더 내 악성 파일
4위 (낮음)   : 외부 직접 침투 — 노출된 표면 자체가 작음
5위 (매우낮음): 텔레그램 봇 통한 침투 — Telegram이 중간 레이어
```

### RPD 결론

> 외부 해커가 이 맥미니를 타깃으로 삼아 능동적으로 공격할 가능성은 낮다.
> 더 현실적인 위협은 **무차별 공급망 공격**과 **본인의 설정 실수**다.

### 현실적 방어 전략

| 위협 | 대응 |
|------|------|
| 공급망 공격 | `npm audit` 주기적 실행, lockfile 고정 |
| 프롬프트 인젝션 | 마운트 범위 최소화, 민감 폴더 제외 |
| .env 탈취 | 1Password 등 시크릿 매니저 활용 |
| 본인 실수 | 마운트는 필요한 폴더만, 파괴적 작업 전 확인 |
| 피싱 | 봇 토큰/OAuth 토큰 절대 공유 금지 |
