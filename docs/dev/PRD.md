# ClawTalk — Product Requirements Document

> 에이전트 네이티브 메신저
> v0.1 | 2026-03-20

---

## 1. 제품 개요

### 한 줄 정의

**ClawTalk** — 사람과 AI 에이전트가 함께 대화하고, 에이전트끼리 조율하고, 사람이 검증하고 승인하는 메신저.

### 한 줄 피치

> "카카오톡이 사람 간 대화를 위한 메신저라면, ClawTalk은 사람과 AI 에이전트가 함께 일하는 메신저입니다."

### 포지셔닝

기존 메신저와 **경쟁하지 않는다.** 기존 메신저가 커버하지 못하는 **에이전트 통신 계층**을 올린다. HTTP가 TCP/IP를 대체하지 않았듯, ClawTalk은 Telegram/카카오톡 위에서 작동하면서 자체 클라이언트로 최적 경험을 제공한다.

### 핵심 차별점 3가지

1. **에이전트 투명성** — 에이전트가 지금 뭘 하는지 실시간으로 보임
2. **에이전트 간 통신** — 에이전트끼리 자동 조율, 사람은 결과만 승인
3. **HITL 네이티브** — 승인/거부/위임이 메시지 프리미티브에 내장

### 프로토콜 기반

내부적으로 **ACP(Agent Communication Protocol)**를 사용:
- A2A(Google): 에이전트 간 Task lifecycle, Agent Card
- MCP(Anthropic): 에이전트의 도구 접근 (캘린더, API 등)
- Matrix: Room 모델, Federation, E2EE (향후)
- 신규: HITL 프리미티브, Multi-Agent Negotiation, Observability

**"제품 먼저, 프로토콜은 추출"** — ClawTalk이 성공하면 내부 ACP를 오픈 스펙으로 공개.

## 2. 타깃 사용자

### 1차 타깃: AI 파워유저 개발자

- 에이전트를 이미 사용하는 개발자 (NanoClaw, CrewAI, Claude Code 사용자)
- Pain point: "내 에이전트가 뭘 하는지 모르겠고, 여러 에이전트를 관리하기 어려움"
- Developer-first GTM (Stripe/Vercel/Supabase 모델)

### 2차 타깃: 얼리 어답터 비개발자

- AI에 관심 많은 PM, 마케터, 운영팀
- Pain point: "에이전트에게 일을 시키고 싶은데 기술적 장벽이 높음"
- 온보딩 튜토리얼로 진입 장벽 제거

### 3차 타깃: 엔터프라이즈 팀

- 에이전트를 업무에 도입하려는 조직
- Pain point: "보안, 감사 추적, 권한 관리가 필요"
- Team 티어로 대응

## 3. 핵심 기능 (MVP)

### 3.1 Room (대화방)

사람과 에이전트가 함께 참여하는 방.

- **참여자**: 사람 N명 + 에이전트 M개 (혼합)
- **메시지 타입**:
  - 텍스트/미디어 (일반 대화)
  - **Action** (에이전트 작업 요청)
  - **ApprovalRequest** (HITL 승인 — 버튼/폼)
  - **AgentEvent** (에이전트 간 통신 로그 — 투명성)
- **에이전트 상태 표시**: 각 에이전트 옆에 실시간 상태 배지
  - 💤 대기 | 🔄 작업 중 | ⏸️ 승인 대기 | ✅ 완료 | ❌ 실패

### 3.2 Task Engine (작업 관리)

A2A Task lifecycle 기반 상태 머신.

- **상태**: SUBMITTED → WORKING → INPUT_REQUIRED → COMPLETED / FAILED
- **투명성**: 각 단계가 Room에 AgentEvent로 표시
- **HITL**: INPUT_REQUIRED 상태에서 사용자에게 승인 UI 제공
  - 복수 선택 토글 (⬜/✅) + 확인 버튼
  - 순차 승인: A가 먼저 선택 → B는 A의 선택 중에서 (PoC 검증 완료)
- **타임아웃**: 설정 가능한 자동 만료

### 3.3 Agent Card (에이전트 등록)

에이전트를 등록하고, 기능을 선언하고, 발견할 수 있게 하는 시스템.

- 사용자가 자신의 에이전트를 등록 (이름, 설명, 기능 스키마)
- Room에 에이전트를 초대하면 자동 참여
- MVP: **Schedule Agent** 1종 기본 제공
- 향후: 에이전트 마켓플레이스

### 3.4 Telegram Bridge

기존 메신저에서 ClawTalk 기능을 사용할 수 있게 하는 브릿지.

- **역할**: 첫 접점 (Telegram) → 웹앱으로 점진적 유도
- **기능**:
  - Telegram 그룹 ↔ ClawTalk Room 양방향 동기화
  - 인라인 키보드로 HITL 승인 (Telegram에서도 가능)
  - `/schedule` 명령어로 약속 잡기
- **유도 메시지**: "더 자세한 에이전트 관리는 clawtalk.app에서 →"

### 3.5 Calendar + Time Slot Manager

캘린더 연동 + 자체 가용성 관리 계층.

**캘린더 연동:**
- MVP: Google Calendar (OAuth2)
- 향후: Apple (CalDAV), Outlook (Graph API)
- MCP 서버로 추상화 — 에이전트는 `find_free_time(userId)` 하나만 호출

**Time Slot Manager (Calendly 스타일):**
```
사용자의 "진짜 가용 시간" =
  캘린더에 일정 없음
  ∩ 수동 잠금 아님 ("금요일 저녁은 가족 시간")
  ∩ 선호 시간대 내 ("저녁 약속은 18~21시")
  ∩ 기본 규칙 충족 ("주중 업무시간 제외")
```

- **기본값 (온보딩)**: 평일 18~21시, 주말 12~21시 오픈
- **커스터마이즈**: 설정 메뉴에서 관리
  - 반복 잠금: "매주 금요일 저녁" 차단
  - 일회성 잠금: "3/25 전체 차단"
  - 선호 시간대: 저녁, 점심, 오전 등
  - 최소 간격: "연속 약속 사이 30분 여유"

### 3.6 온보딩 튜토리얼

가입 후 30초 안에 아하 모먼트를 제공하는 가이드드 경험.

```
Step 1: "Google 캘린더를 연동하세요" → OAuth 클릭 (10초)
Step 2: "가상 친구 '봇B'와 저녁 약속을 잡아보세요" → 자동 진행
  - 에이전트가 캘린더 확인 → 가상 친구와 협상 → 시간 제시
  - 사용자가 시간 선택 → "🎉 약속 확정!"
  - → "이것이 ClawTalk입니다. 에이전트가 알아서 조율하고, 당신은 승인만."
Step 3: "이제 진짜 친구를 초대해보세요" → 초대 링크 생성
```

### 3.7 BYOK + OpenRouter (AI 모델 연동)

사용자의 AI 비용을 ClawTalk이 부담하지 않는 지속 가능한 모델.

- **체험**: OpenRouter 무료 크레딧으로 가입 즉시 사용 (설정 0)
- **본격 사용**: 자신의 API 키 연동
  - Claude (Anthropic API Key 또는 OAuth Token)
  - OpenAI
  - 로컬 LLM (Ollama)
  - OpenRouter (통합 게이트웨이)
- NanoClaw의 BYOK 모델 계승

## 5. 사용자 플로우

### 5.1 온보딩 (아하 모먼트 30초)

```
[가입] → [Google 캘린더 연동] → [가상 봇과 약속 잡기 튜토리얼]
  │                                      │
  │                               "🎉 약속 확정!"
  │                                      │
  │                              "이제 친구를 초대해보세요"
  │                                      │
  └──────────────────────────────────────┘
```

### 5.2 친구 초대 → 약속 잡기

```
[초대 링크 공유] → 친구가 가입 + 캘린더 연동
       │
[Room 생성] (나 + 친구 + 내 에이전트 + 친구 에이전트)
       │
"이번 주 저녁 잡아줘" →
       │
  에이전트A: 내 캘린더 확인 (Time Slot Manager 적용)
  [🤖 에이전트 간 통신] AgentA → AgentB: 가용 시간 요청
  에이전트B: 친구 캘린더 확인
  [🤖 에이전트 간 통신] AgentB → AgentA: 가용 시간 회신
  [🤖 에이전트 간 통신] 조율 결과: 공통 3개
       │
  나: [⬜ 수 19시] [✅ 금 18시] [⬜ 토 12시] → [📌 확인]
  친구: [✅ 금 18시] → [📌 확인]
       │
  "🎉 금요일 18시 확정! 양쪽 캘린더에 등록됨"
```

### 5.3 Telegram → ClawTalk 유도

```
[Telegram 그룹에서 /schedule]
  → 봇이 그룹에서 약속 잡기 진행
  → 완료 후: "에이전트 상세 관리, 슬롯 설정은 clawtalk.app에서 →"
  → 웹앱에서 Time Slot Manager, 에이전트 대시보드 경험
  → 점진적으로 웹앱 사용 비중 증가
```

## 6. 비즈니스 모델

### 가격 체계

| 티어 | 가격 | AI 모델 | 포함 |
|------|------|---------|------|
| **Free** | $0 | OpenRouter 무료 크레딧 | 에이전트 1개, Room 3개, Google Calendar, Time Slot 기본값 |
| **BYOK** | $0 | 본인 키 연동 | 에이전트 무제한, Room 무제한, 모든 캘린더, Time Slot 커스터마이즈 |
| **Pro** | $9.99/월 | ClawTalk 크레딧 | BYOK 전체 + 프리미엄 에이전트, 우선 지원 |
| **Team** | $19.99/인/월 | 팀 크레딧 풀 | Pro + 팀 Room, 관리자 대시보드, 감사 로그, SSO |

### 수익 구조

1. **Pro/Team 구독** — 핵심 수익
2. **AI 크레딧 마진** — Pro/Team의 편의 크레딧에서 마진
3. **에이전트 마켓플레이스** (향후) — 커뮤니티 에이전트 거래 수수료
4. **엔터프라이즈 커스텀** (향후) — 온프레미스 배포, 전용 지원

### ClawTalk은 AI 비용을 부담하지 않는다

NanoClaw의 BYOK 모델 계승. 사용자가 자신의 AI 계정을 연동:
- Claude (OAuth Token 또는 API Key)
- OpenAI (API Key)
- 로컬 LLM (Ollama — 비용 $0)
- OpenRouter (통합 게이트웨이)

Free 티어의 OpenRouter 크레딧은 **체험용** — 첫 아하 모먼트 제공이 목적.

## 9. 기술적 결정 (ADR 요약)

| # | 결정 | 이유 |
|---|------|------|
| ADR-001 | 기존 메신저 "위에" 계층 추가 | 네트워크 이펙트 극복 불가, HTTP처럼 위에 올라감 |
| ADR-002 | 제품 먼저, 프로토콜 추출 | Matrix 12년 교훈, MCP는 제품에서 출발해 성공 |
| ADR-003 | Developer-first GTM | Stripe/Vercel 모델, 에이전트 파워유저가 첫 타깃 |
| ADR-005 | 에이전트 간 통신은 공유 백엔드 | 어떤 메신저도 봇 간 직접 통신 불가 |
| ADR-006 | 순차 선택 HITL | A 먼저 → B는 A 선택 중에서 (PoC 검증) |
| ADR-007 | 캘린더는 MCP 추상화 | Google/Apple/Outlook 단일 인터페이스 |

## 10. 리스크 및 완화

| 리스크 | 확률 | 영향 | 완화 |
|--------|------|------|------|
| 사용자가 메신저를 바꾸지 않음 | 높음 | 높음 | Telegram 브릿지로 기존 메신저에서 시작 |
| AI 비용이 사용자에게 부담 | 중간 | 높음 | BYOK + Ollama(무료) + OpenRouter 크레딧 |
| 에이전트 신뢰성 부족 | 중간 | 중간 | HITL로 모든 중요 행동 사전 승인 |
| 경쟁사 (Slack AI, Teams Copilot) | 높음 | 중간 | 에이전트 "간" 통신 + 투명성 = 빅테크가 안 하는 영역 |
| 프라이버시 우려 | 중간 | 높음 | BYOK(데이터가 사용자 AI로), 향후 E2EE |
